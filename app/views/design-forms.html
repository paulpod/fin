Forms

Forms should be logically laid out and read from top to bottom.

Fields should clearly be defined with a 2 pixel border.

Fields should have a label, positioned above the field.

Examples of what kind of input is required can go below the field, or use this to replay calculations to the user.

Current price: 1695.4

Min 1, max 100


Form fields that can use a numeric input should have the HTML setting to use a number pad on mobile. Use type="number" as an attribute of the input element to show the appropriate keypad for data entry on mobile devices.

Example: 1233.33

Similarly fields that are dates can use native date pickers by including type="date" as an attribute.

Example: 10/07/2016

Some form fields not editable, but might become editable if some state changes (eg. price moves into range, or funds are provided).

It is useful to display these in a state that communicates this 'input field, but not yet' potential, and also to reserve the appropriate space in the layout.

Use the disabled="disabled" attribute on the input element to stop it being selected or tabbable.
