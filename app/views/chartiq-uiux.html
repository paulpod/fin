ChartIQ implementation notes

We reviewed the default pages included with the ChartIQ package and there are a number of UX improvements from the default we would like to implement before launch. Most of these are specific to the phone implementation, which is just a squashed version of the full-size chart really.

Chart menu

On mobile devices, this popup menu is too long to display well. Suggest using a new page, or full page overlay in the style of a standard list to pick from the long list.

Example menu page, as HTML+CSS closely following iOS styles.


Indicators menu

On mobile devices, this popup menu is also too big and complicated. Suggest using a set of new pages, to pick from and apply the relevant settings for the indicator on an individual mobile OS style page. Pressing 'Create' for each should add to chart, and return user to chart view.

Example menu page, as HTML+CSS closely following iOS styles.


I have used the Framework7.io library to achieve these iOS-style pages in HTML+CSS, but other frameworks can achieve similar results. A set of Android/Material Styles are included too. Alternatively, if native can be integrated with the HTML charting that might work too.

Note: Sliders don't appear when viewed with a desktop browser. For best results visit links on a phone.
